04 —배경: Beer-Lambert 법칙

RGB = illuminant × exp(−OD)
log(RGB) = log(illuminant) − OD
  • log 도메인에서 조명은 additive offset
  • OD = −log(RGB): 조명 변화 → 전체적 평행이동만 발생
  • 채널 비율 log(R/G), log(G/B): 중성 조명 가정 시 조명 독립
  • InstanceNorm / channel norm → 조명 offset 제거 가능
# ODPreprocess: RGB → [OD × 3ch, log(R/G), log(G/B)] → 1×1 Conv → 3ch

08 —FrequencyGate — 핵심 모듈

# 1. FFT + 저주파 중앙 정렬 X_f = torch.fft.fft2(x, norm='ortho') X_f = torch.fft.fftshift(X_f, dim=(-2, -1)) # 2. 반경 기반 마스크: r = dist / (min(H,W)/2) low_m = (r < 0.1).float() # 조명 억제 대역 mid_m = (r 사이 0.1~0.4).float() # chromophore high_m = (r >= 0.4).float() # wrinkle/texture # 3. Learnable gates (초기값 물리 기반) g_low = torch.sigmoid(self.low_logit) # init≈0.12 (억제) g_mid = torch.sigmoid(self.mid_logit) # init≈0.88 (보존) # 4. 가중치 적용 + iFFT X_chroma = X_f * (low_m * g_low + mid_m * g_mid) x_chroma = ifft2(ifftshift(X_chroma)).real
  • low_logit 초기값 −2: sigmoid ≈ 0.12 → 조명 대역 처음부터 억제
  • 학습으로 최적 대역 자동 발견 — 고정 필터가 아님
  • low_gate 값이 학습 중 0에 가까워질수록 조명 분리 성공

09 —ODPreprocess — 물리 기반 입력 변환

Optical Density (OD)
OD = −log(RGB + ε)
Beer-Lambert 기반
조명이 additive offset → 정규화로 제거 가능
Channel Log-Ratio
log(R/G), log(G/B)
중성 조명 가정 시
조명에 완전 독립적인 표현
[OD×3, log(R/G), log(G/B)]
→
Conv 5→3 (1×1)
→
[B, 3, H, W]
  • 5ch → 3ch: 학습 가능한 압축 — 최적 표현 자동 발견
  • 초기값: OD 채널 그대로 통과 (identity init) → 안정적 수렴
  • FrequencyGate와 상호보완: 입력 + feature 두 단계에서 조명 제거

10 —Loss 전략 — 3단계 학습

PhaseEpoch활성 Loss목적
Phase 1 0 ~ 40% L1(brown, red, wrinkle) + Beer-Lambert Recon 빠른 수렴, 기본 분리 학습
Phase 2 40 ~ 80% + Illumination Consistency 조명 augment 이미지와 동일 출력 강제
Phase 3 80 ~ 100% + FreqGate Regularization low_gate → 0 유도, 조명 억제 강화
# Illumination Augmentation: 밝기 × 0.5~1.8, 색온도 × 0.7~1.3
# 틴트 편향, 공간 Vignetting (30% 확률)
loss_consist = L1(brown_orig, brown_aug) + L1(red_orig, red_aug) + ...
  • Beer-Lambert Recon: OD → RGB 복원으로 물리 제약 부여 (w=0.3)
  • FreqReg: sigmoid(low_logit).mean() → 0 유도로 조명 대역 억제 보장

11 —기존 연구 대비 차별점

vs DINOv2 기반
  • DINOv2: 대형 ViT, 86M+ 파라미터
  • FreqAwareUNet: 경량, 도메인 특화
  • 물리 기반 설명 가능 (Explainable)
  • 주름 분석: High-freq 자연스러운 분리
vs 기존 Retinex
  • Retinex: 전처리로만 적용 (정적)
  • FreqGate: 네트워크 내부에서 동적 분리
  • 대역 경계를 데이터에서 자동 학습
  • Task별 최적 주파수 자동 발견
vs 일반 U-Net
  • 일반: 조명 정보 그대로 skip
  • FreqAware: 주파수 대역별 task 라우팅
  • 주름: high-freq만 사용 → 조명 음영과 분리
  • 단일 모델로 3가지 task 동시 처리
3중 이론 융합
  • Beer-Lambert: OD 도메인 변환
  • Retinex: 조명 = 저주파
  • Deep Learning: 최적 대역 학습
  • → 논문 기여로 출판 가능한 조합

SUMMARY

결론 및 향후 계획

  • FrequencyGate: Skip connection에서 FFT 기반 주파수 대역 분리 — 논문 첫 시도
  • ODPreprocess: Beer-Lambert 물리 법칙 기반 조명 불변 입력 변환
  • Multi-task: Brown, Red, Wrinkle 동시 예측 — 단일 순방향 패스
  • 3단계 학습: Supervised → Consistency → FreqReg 순차 활성화
